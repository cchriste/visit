VisIt 1.1.4 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.1.4

  • VisIt's plot list has been enhanced so that you can expand plots by clicking on the turndown at the start of the plot name. Once a plot is expanded, you can move operators up or down in the pipeline as well as remove them from the middle of the pipeline. You can also open the plot attribute or operator attribute windows by double clicking on a plot or operator name once the plot list entry is expanded. You can also activate the Subset window by double clicking on the plot's subset icon.
  • VisIt has been modified so that you only need to enter your password for remote computers one time. VisIt launches a new component launcher on the remote computer and the launcher takes care of launching the metadata server and the compute engine.
  • VisIt has a new Boundary plot that is based on the Subset plot.
  • The Silo database reader plugin has been improved so that it can open large files much faster.
  • VisIt now always passes the desired version when launching components. This gets around problems where a network may have different versions of VisIt.
  • Multiple coordinates are no longer required for orthogonal slices.
  • Closing a compute engine is faster now.
  • The GUI's -o option to open files on the command line now opens a virtual database if the file being opened has related timesteps and automatic file grouping is on. This lets you double click on any file in a family of files and have VisIt automatically open the file family, which makes it easier to change timesteps later.
  • Picking no longer requires recalculation when slices are projected to 2D.
  • The reflect operator now has a new default reflect octant.

Bugs fixed in version 1.1.4

  • Accidentally clicking the "All" buttons in the Subset window causes the GUI to crash when the window has no subsets.
  • Makefiles for generated plugins are incorrect.
  • View autocentering has been replaced with "Maintain view" and it is off by default. This ensures that the view recalculates when it needs to unless you are maintaining the view.
  • The 2D axes sometimes have an extra line.
  • Pick reports 2D pick points when picking on 3D meshes.
  • Invalid host profiles are kept in the config file.
  • Lineout does not work with material species.
  • VisIt does not run on Windows NT 4.0.
  • The keyframe regression test crashes the viewer.
  • The placement of the Ok and Cancel buttons has been changed in the File selection window.
  • Deleting curves causes plots in other windows to be deleted.
  • VisIt's viewer sometimes crashes when it loses its connection to a metadata server.
  • VisIt can now start up even if it cannot launch a metadata server.
  • 2D slice view is not set correctly with empty datasets.
  • VisIt now complains if you try to use an invalid color table.
  • The Connection progress dialog is not brought to the front on Windows.
  • Confusion regarding pick of Subset plot of domains.
  • Pick reports incorrect node coordinates for projected slices.
  • The Point3D database reader gets an extra point.
  • Window limits don't adjust as we step through an animation.
  • Curve view settings are wrong for a lineout of a constant plot.
  • The Slice operator's flip option should not be available when projecting the slice to 2D.
  • The mdserver assumes the wrong path when a file is given using the -o option on startup.
  • The viewer reports invalid information for parallel compute engines launched via the command line.
  • The Host profiles window has been moved to the Options menu.
