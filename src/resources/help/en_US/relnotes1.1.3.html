VisIt 1.1.3 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.1.3

  • VisIt now supports automatic file grouping so you don't need a .visit file in order to view time-varying data using the Main window's VCR controls. File grouping is turned on/off in the File Selection Window. File grouping is now on by default so large sets of data files should appear as a single option in the window. If you see files that you feel do not belong in a database that has been created through automatic file grouping, you can change the filename filter to try and remove the unwanted files. If you turn off automatic file grouping, files that could otherwise be grouped are available independently as before.
  • Host profiles now have a common name that can be used to identify remote machines. The common name is used to tell VisIt which remote machine to log into but it can also use a list of alias names, which can contain wildcards, for that machine.
  • The Host Profiles window has been modified to make better use of space. All of the parallel options are on a second tab that is only enabled if the active host profile is parallel.
  • VisIt now allows the use of PSUB to launch parallel compute engines on parallel Linux clusters.
  • VisIt now notifies the user when it needs to re-execute a plot network when picking.
  • VisIt now has default host profiles for launching compute engines on emperor, adelie, pengra, edgewater, and whitecap.
  • VisIt allows 2d linear curves to be saved as ULTRA files.
  • VisIt has new data queries for compactness in the Queries window.
  • VisIt writes better quality output to its curve format output files.
  • The Lighting window now has a spinbox (in addition to the old slider) that lets you set the light brightness.
  • The popup menu in VisIt's vis windows now has options in the Customize menu to turn toolbars on and off for all vis windows.
  • VisIt's plugin creation tools now support database plugins for databases with multiple domains and multiple timesteps.
  • VisIt now has a default value of "localhost" for the -launchengine command.

Bugs fixed in version 1.1.3

  • VisIt could not handle .visit files without absolute paths on the Windows operating system.
  • VisIt crashes when creating a Subset plot of bigsil.silo.
  • VisIt crashes when creating a new plot with "Replace plots mode" enabled.
  • The Tube operator does not with on the springs.dat database.
  • Autocloning a window messes up lineouts.
  • VisIt crashes when launching a compute engine and telling it to read certain bad Silo files.
  • VisIt could not produce animations of defvars stored in Silo files because the defvar was not recognized as a valid variable for timesteps other than the first timestep.
  • VisIt is too aggressive about opening files causing lots of unnecessary work when opening files..
  • Pick was malfunctioning when running VisIt with a parallel compute engine.
  • The toolbar in curve windows is enabled when it shouldn't be enabled.
  • When VisIt is used to plot two unrelated time-varying databases in the same window, the number of timesteps used for each database can be wrong in keyframing mode.
  • When the vis window is shrunk to its smallest size with toolbars enabled, some areas of plots are rendered out of sight.
  • The Box operator has occasional problems with point meshes.
  • The password window is obscured by the splashscreen on the Windows platform.
  • Subset plots of points fail.
  • Problem with zoneno in face lists with 1-origin.
  • Vectors do not play well with "actual" spatial extents.
  • VisIt prints out various debug messages to the console.
  • Pick points don't appear if the network that generates a plot has to re-execute.
  • The code generation tool XMLEdit messes up enables that use enums.
  • Picking re-executes networks too often when data has been transformed.
  • VisIt can't onion peel a Subset plot.
  • Example Python scripts are out of date.
  • Mesh plots of point meshes turn to outlines after clicking the Mesh plot window's Apply button.
  • The -foreground, -background, -style options are broken.
  • The Silo database reader makes too many initialization calls.
  • VisIt allows parallel tasks of one processor.
