VisIt 2.13.1 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Sections

  • Bug Fixes
  • Enhancements
  • Changes for VisIt developers

Bugs fixed in version 2.13.1

  • Modified the query over time to store the results in double precision instead of single precision.
  • Modified the VTK Reader to ensure material scalars in multi-block datasets are read correctly.
  • For Windows platform, VisIt will now display an error message if VISITSSH defined, but does not point to an executable, and if a parallel engine is requested but mpiexec.exe cannot be found.
  • Fixed the issue with the ScatterPlot Setup Wizard populating the variable menus from a previous database.
  • Fixed the thresholding for point meshes.
  • Fixed crash performing a lineout when -dump issued on command line.
  • VisIt's viewer was unnecessarily creating text labels that would likely not be used unless using a ParallelCoordinates plot. For Subset plots with large numbers of domains, this caused the viewer to use many gigabytes of memory even when scalable rendering was used.
  • Fixed crash performing a time-query when Curve plot is disabled. An error message will now be issued, and the query will not be performed.
  • Fixed build_visit build of Qt with -no-openssl on Fedora.
  • Set MetadataIsTimeVarying in exported Silo databases by fiat. This is a pre-caution to ensure we handle exported data correctly in cases where metadata really is time-varying and probably a minimal performance hit in cases where its not necessary.
  • Fixed crash opening certain netcdf-4 files on Windows.
  • Fix crashes when any single block has number of nodes/zones or tuples x number of components that exceeds INT_MAX.
  • Fix Save Window compression options so that only those options appropriate to currently selected format are enabled for selection.
  • Fixed a bug with the active window selector in the main window so that it now displays more than a single digit if necessary.
  • Fixed problem running PythonFilters in parallel on Windows.
  • Fixed hang starting VisIt 2.13.x on Windows VM.

Enhancements in version 2.13.1

  • Enhanced the SPCTH reader to process a list of MTMD files as one MTMD file.
  • Updated the parallel interactive pdebug host profile for LLNL cab machine to handle job launching changes on the machine.
  • Updated the parallel viz host profile for the LANL trinity machine to handle job launching changes on the machine.
  • Enhanced Pixie plugin to read datasets with more than INT_MAX nodes/zones.
  • Default compression method for Save Window was changed from PackBits to None.

Changes for VisIt developers in version 2.13.1

  • Modified build_visit to pass the zlib directory information to the configure command for hdf5 when -zlib was specified on the command line.
  • Modified bv_pyside.sh to patch pyside2 that removes the conditional collecting of the Multimedia, WebEngineWidget, and WebKit modules.
  • Fixed the QWT make install symlink error for OSX.
  • Fixed the PySide2 and Shiboken2 library naming issue for OSX.
  • Changed build_visit to build Qt in silent mode by default. Added the --no-qt-silent argument to build_visit that disables building Qt in silent mode.
  • Changed when default variable units are set on the viewer, in order that plot plugins can override the default values in their CustomizeBehavior method.
  • Upgraded numpy from 1.11.2 to 1.14.1.
