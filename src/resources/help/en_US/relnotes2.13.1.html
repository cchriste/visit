VisIt 2.13.1 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Sections

  • Bug Fixes
  • Enhancements
  • Changes for VisIt developers

Bugs fixed in version 2.13.1

  • Query over time was modified to store the results in double precision instead of single precision.
  • The VTK Reader was modified to ensure material scalars in multi-block datasets are read correctly.
  • VisIt will now display an error message on Windows if VISITSSH is defined, but does not point to an executable, and if a parallel engine is requested but mpiexec.exe cannot be found.
  • A bug was corrected with the ScatterPlot Setup Wizard populating the variable menus from a previous database.
  • A bug was corrected with the Threshold operator thresholding point meshes.
  • A bug was corrected where VisIt would crash when performing a lineout when -dump was specified on the command line.
  • VisIt's viewer was enhanced to address a performance issue where it unnecessarily created text labels that would likely not be used unless displaying a ParallelCoordinates plot. For Subset plots with large numbers of domains, this caused the viewer to use many gigabytes of memory even when scalable rendering was used.
  • A bug was corrected where VisIt would crash performing a time-query when the Curve plot was disabled. An error message will now be issued, and the query will not be performed.
  • A bug was corrected with build_visit with the building of Qt with -no-openssl specified on the command line on Fedora systems.
  • The exporting of data to Silo files was modified to always set the MetadataIsTimeVarying attribute to ensure VisIt handles exported data correctly in cases where the metadata is time-varying. This will probably result in a minimal performance degradation in cases where its not necessary.
  • A bug was corrected where VisIt would crash opening certain NetCDF-4 files on Windows.
  • A bug was corrected where VisIt would crash when the number of nodes or zones or tuples times the number of components exceeds INT_MAX in a single block.
  • A bug was corrected with the Save Window compression options so that only those options appropriate to the currently selected format are enabled for selection.
  • A bug was corrected with the active window selector in the main window so that it now displays more than a single digit if necessary.
  • A bug was corrected with running Python filters in parallel on Windows.
  • A bug was corrected where VisIt would hang on start up when running on a Windows VM.
  • A bug was corrected with Python filters failing on Windows 8, 10.
  • A bug was corrected where the VisIt metadata server would crash when it encounterd a Silo file with a multimesh that was invalid, such as when the number of blocks specified was zero.
  • VisIt was modified so that crash session files created by any running VisIt process are correctly ignored by any additional VisIt processes that are launched.

Enhancements in version 2.13.1

  • The SPCTH reader was enhanced to process a list of MTMD files as one MTMD file.
  • The parallel interactive pdebug host profile for the LLNL cab machine was modified to handle job launching changes on the machine.
  • The parallel viz host profile for the LANL trinity machine was modified to handle job launching changes on the machine.
  • The Pixie reader was enhanced to read datasets with more than INT_MAX nodes or zones.
  • The default compression method for saving windows was changed from PackBits to None.
  • The performance of VisIt was improved when populating file lists from directories with large numbers of entries.
  • VisIt now saves crash session files for a specific VisIt process. The user can choose which crash session file to restore if multiple ones exist.
  • VisIt has been enhanced to select between chaos 5 and toss 3 executables on LLNL systems based on the machine name instead of the environment variable SYS_TYPE. Some users didn't have the SYS_TYPE environment variable set causing the previous logic to fail in some instances.

Changes for VisIt developers in version 2.13.1

  • Build_visit was modified to pass the zlib directory information to the configure command for hdf5 when -zlib was specified on the command line.
  • Build_visit was modified to patch pyside2 to remove the conditional collecting of the Multimedia, WebEngineWidget, and WebKit modules.
  • A bug was corrected with the creation of a QWT symbolic link on OSX.
  • A bug was corrected with the naming of the PySide2 and Shiboken2 libraries on OSX.
  • Build_visit was modified to build Qt in silent mode by default. Additionally, build_visit was modified to allow specifying --no-qt-silent on the command line to disable building Qt in silent mode.
  • The location where the default variables are set in the viewer was changed so that plot plugins can override the default values in their CustomizeBehavior method.
  • Numpy was upgraded from version 1.11.2 to 1.14.1.
  • The Python CLI was enhanced to make it possible to the control visibility of annotation objects during their creation with an additional argument in the CreateAnnotationObject() method.
  • Make package was modified so that it no longer includes any directories named icons. Previously, this resulted in a couple of invalid symbolic links being added to the distribution file.
  • Uintah was upgraded from version 2.1.0 to 2.5.0 beta.
  • GDAL was upgraded from version 1.10.0 to 2.2.4.
  • Conduit was upgraded from version 0.3.0 to 0.3.1.
  • Build_visit was modified to patch BoxLib so that it would compile with newer versions of GCC.
