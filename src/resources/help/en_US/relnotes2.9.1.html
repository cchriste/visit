VisIt 2.9.1 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Sections

  • Bug Fixes
  • Enhancements
  • Changes for VisIt developers

Bugs fixed in version 2.9.1

  • The meta data server no longer crashes when it attempts to open a Silo file with empty multivar variables.
  • Pick of expression variable that requires pipeline re-execution no longer causes an engine crash/hang in parallel.
  • VTK Mapper has been updated to use a Log-scale, if selected, for texture coordinates and scaling when applying color using textures.
  • Scatter Plot now supports double-precision input.
  • Picking on plot with CoordSwap operator will no longer trigger Pipeline re-executions unless something else in the pipeline requires it.
  • Nek5000 data files can now be opened correctly on Windows.
  • OnionPeel on a Subset Plot of AMR data has been fixed.
  • PFLOTRAN reader no longer causes the engine to crash when reading vector data.
  • Threshold operator attributes dialog has been updated to display small min/max field values correctly
  • The color-using-textures rendering preference is now preserved when settings are saved.
  • The Data-Level Comparsion wizard now sets the variable-type of the expression being created.
  • The 3D tab in the Annotations window has been modified so that the bounding box settings can be modified even when the axes portion is turned off.
  • A bug was fixed where Plot/Operator windows accepting doubles as input would truncate and/or round numbers being entered, thus possibly causing loss of significant digits. Occurred generally for very small numbers.
  • Fixed a glitch with the Expression creator window, where text could be parsed as html, eg <P> became new paragraph.
  • Created a generic ExchangeVector method to call the correct Exchange*Vector method based on the underlying data type during ghost zone communication.

Enhancements in version 2.9.1

  • A parallel mxterm launch profile was added to many of the LLNL host profiles. Using the parallel mxterm launch profile will allow users to easily run parallel jobs when launching VisIt from within an mxterm.
  • VisIt's XDMF reader can now read XDMF files that consist of a top level Grid collection containing intermediate grid collections and each of those contains a single grid.

Changes for VisIt developers in version 2.9.1

  • The --nektar++ option to build_visit was renamed to --nektarpp to avoid problems with some shells in dealing with the ++ in function names.
