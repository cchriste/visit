General features added in version 2.12

Advanced features added in version 2.12

  • Added experimental merge_tree, split_tree, and local_threshold serial-only expressions for segmentation using the topology of a scalar field. The merge_tree and split_tree expressions take an input scalar and create a new scalar field with segmentation labels. The local_threshold expression takes a scalar field and a segmentation label and applies a local threshold metric to create a new scalar field.

Changes in GUI behavior for version 2.12

File format reader changes in version 2.12

  • The Cosmos++ database reader has been updated with new features and bug fixes.
  • Fixed node order problem with hex27 elements in Exodus files.
  • Several of the ASCII-based database readers (PlainText, Curve2D, Lines, Rect, etc.), were upgraded to support on-the-fly decompression via gzstream class. This means VisIt can directly read a gzip'd input file in these formats. Previously, this would have been handled through VisIt's ZipWrapper database reader but this new apporach is simpler and more efficient for simple ASCII-based files. This capability does not currently extend to ASCII-based formats that are read via other libraries such as VTK, MFEM, etc. In addition, as part of this enhancement VisIt's Save Window feature can now save gzip'd compressed curve formats. In this case, the quality slider selects the gzip level in the range [1..9]. Some of the ASCII Export Database formats were also enhanced.

Changes to VisIt's plots in version 2.12

Changes to VisIt's plots in version 2.12

  • Corrected a bug in the Pseudocolor plot where displaying lines from the IntegralCurve operator as tubes could cause some of the lines to disappear.

Changes to VisIt's picks and queries in version 2.12

  • Fixed the issue with the XRay Image Query crashing when given bad variable names.

Other bugs fixed in version 2.12

  • Fixed the GDAL build_visit error for OSX 10.11
  • Simulations instrumented with Libsim using a batch-style integration would crash when evaluating expressions.
  • Simulations instrumented with Libsim using a batch-style integration were not able to use operator-created variables.
  • Exporting streamlines caused the engine to crash because it was trying to save integral curve objects that no longer existed.
  • Fixed crash running parallel python simulation examples.

Configuration changes in version 2.12

  • VisIt was ported to and installed on Los Alamos National Laboratories' Trinity system. A host profile was added for running in client/server mode from Lawrence Livermore National Laboratory.

Build changes in version 2.12

  • build_visit was enhanced to build version 1.2.4 of the SeedMe Python Client.
  • Xdmf configuration in build_visit was modified to correctly set XDMF_SYSTEM_ZLIB and XDMF_SYSTEM_LIBXML2 to ON, since we use the vtk versions during the build.
