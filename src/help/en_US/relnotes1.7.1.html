VisIt 1.7.1 Release Notes

Welcome to VisIt's release notes page. Release 1.7.1 is primarily a bug fix release to release 1.7. Look here to see a complete list of bugs fixed in release 1.7.

Bugs fixed in version 1.7.1

  • The slice operator's project feature was not correctly projecting the slice to the plane Z=0. This bug was exhibited only with data sets that had very large spatial extents.
  • Apply either the Displace operator or Transform operator followed by the Box operator would sometimes result in missing data.
  • The Chombo reader used 32-bit integer arithmetic to calculate start offsets for boxes/patches, which would result in errors when loading large files. Calculations now use 64-bit integer arithmetic.
  • The plugin manager was incorrectly loading non-plugin files in some cases.
  • The modules for the Java client have been updated to reflect changes made in release 1.7.
  • Valid printf-style format strings for Label plot and Queries were incorrectly being deemed invalid on Mac.
  • Extents trees were temporarily disabled in Silo plugin to work around issue with Ale3d data files.
  • The Nek3D reader was enhanced to handle parallel, ascii, and 2D variants of the format.
  • The SAS reader was enhanced to expose variables for assembly type and id.
  • A bug manifesting as a C++ runtime error was fixed for the windows version.
  • The NZQH gradient was modified to avoid precision errors on AIX.
