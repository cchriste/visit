VisIt 1.0.2 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.0.2

  • A Java library interface to VisIt has been added. The library allows VisIt to be launched and controlled from Java programs.
  • Material interface reconstruction is faster.
  • Material interface reconstruction now skips over materials that are not used within a domain.
  • A new window has been added to prompt for the number of processors when launching VisIt's compute engine.
  • The Erase operator is much faster.
  • Slicing rectilinear meshes is much faster.
  • The CLI has new methods for setting viewer caching and animation properties.
  • The Onion peel operator supports logical indices.
  • The Lighting window now indicates which lights are active with icons.

Bugs fixed in version 1.0.2

  • Running visit -debug 6 crashes.
  • Pick does not correctly report the picked zone number.
  • Subset plots of domains with material selection applied are sometimes broken.
  • Opening a new files causes plots to be recalculated when pipeline caching is turned off.
  • The number of subsets in the Subset plot window gets multiplied sometimes.
  • Material interface reconstruction gets confused when ghost zones are repeatedly communicated.
  • Splashscreen processes do not exit when VisIt crashes.
  • Sometimes the GUI crashes on exit.
  • The camera light comes on when it is not wanted.
  • Material selection is applied when it is not wanted.
  • SIL restrictions are sometimes applied to plots with an incompatible SIL.
  • Expressions do not work in the CLI.
  • Subset plots of species have been removed.
  • "Selected files" are always the ones in the current directory.
  • The Lighting window's controls have no effect.
  • The Subset plot's legend can get corrupted.
  • Pick should report domain number when the database has multiple domains.
  • Material interface reconstruction always tetrahedralizes clean cells.
  • The Lighting window's direction text field should be enabled for Camera lights.
  • Disable setting the direction for Ambient lights.
  • Rename Direction and Eye lights to something better.
  • The Lighting window does not accept new directions when "Auto update" mode is on.
