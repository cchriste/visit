VisIt 1.2.8 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.2.8

  • VisIt allows you to pick a center of rotation so rotating plots when you are zoomed in does not cause them to whiz off of the screen. To use this option, click on the Choose center button in the vis window's toolbar or popup menu and click on a point on the plot. The picked point will be the new center of rotation and will always appear on the screen.
  • VisIt now provides a MinMax query that can determine the minimum and maximum values from a database; not just the selected plot.
  • VisIt now provides queries that allow you to determine the number of zones or nodes in a mesh.
  • VisIt now allows you to perform axis-aligned lineouts. If you press the Shift key when doing a lineout, the line will be aligned to the horizontal or vertical axis, depending on the slope of the line.
  • VisIt is now installed with the Boxlib database reader plugin on the SCF.
  • VisIt no longer saves the remote host in your config file so it will not try to automatically gain access to a remote computer at startup.
  • The SAMRAI, Tetrad, and PDB database reader plugins are now available for the Windows version of VisIt.
  • VisIt now checks for bad material values so it is easier to determine if a data file was not written correctly.
  • Visit now issues a warning when an invalid material is encountered.
  • The SAMRAI and Tetrad database reader plugins are now build on mcr on the OCF.
  • VisIt's cursor now changes to crosshairs when entering pick mode.
  • Base index is now set up for Boxlib data files.
  • The Mesh plot can optionally now draw all cell edges.
  • VisIt now tells its compute engines to clear their caches when you restore a session file.
  • VisIt's clipping code now handles 2D cell types.
  • VisIt's Compute engines window now has a button to tell the active compute engine to clear its caches. Press this button if your computer becomes sluggish and the compute engine will free up some memory.
  • You can now open Boxlib data files by opening a "Header" file.
  • All variables are now available from Boxlib data files.
  • VisIt's config files are now readable by the Windows notepad program.

Bugs fixed in version 1.2.8

  • VisIt's parallel compute engine can hang when changing time steps.
  • VisIt returns the wrong material list for individual Flash files when the list of materials has to be inferred from the ireg array. This bug caused subset selection to vary unexpectedly over time and it also caused the compute engine to crash when creating animations involving the FilledBoundary plot.
  • Material selection revealed an internal error message.
  • Certain expressions crashed VisIt's compute engine.
  • The IsoVolume operator can crash VisIt's compute engine.
  • Applying the IsoVolume operator and then applying the IsoSurface operator crashed VisIt's compute engine.
  • VisIt hangs on startup when old configuration files are present.
  • Plots disappear when animating over time when making movies.
  • VisIt saves the remote host into the config file.
  • The nmats expression does not return the right values.
  • VisIt's qtssh program, which is used to log into remote computers on Windows, crashes when the Cancel button is clicked.
  • VisIt's GUI is very sluggish with 2000+ variables in the variables list.
  • The matvf expression is not calculated correctly.
  • VisIt does not read all of the variables in a VTK file.
  • The Integrate query for area under a curve returns the wrong value.
  • Zones are missing in the FilledBoundary plot.
  • The OnionPeel operator should not display or work on domains that are not in the current SIL.
  • Subset categories are not immediately available in the OnionPeel operator window.
  • The OnionPeel operator window does not have the correct initial values.
  • Pick indicator is way far away from the pick point.
  • When a single file is in the Selected files list, VisIt's GUI shows the file's entire path.
  • VisIt's compute engine crashed when trying to read an Ensight data file.
  • Fullframe mode requires another expose event to fully activate.
  • Pick problems with high aspect ratio zones.
  • Non-square geometries are not honored by VisIt's CLI.
  • The Surface plot does not have the right view in scalable rendering mode.
  • The legend for Pseudocolor plots is incorrect when Skew scaling is used.
  • The nmats expression does not work properly with the FilledBoundary plot.
  • VisIt does not save its global toolbar visibility setting into the config file.
  • The Windows version of VisIt uses the wrong file name when saving config files.
  • The Password window's Cancel button is too close to the password text field.
  • The wrong warning message is issued when a remote engine launch is cancelled.
  • The View window's rotz command has numerical errors.
  • The matvf expression always returns 0 for some Boxlib data files.
