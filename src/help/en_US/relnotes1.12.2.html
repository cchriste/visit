VisIt 1.12.2 Release Notes

Welcome to VisIt's release notes page. Release 1.12.2 is primarily a bug fix release to release 1.12. It fixes additional bugs in release 1.12. Look here to see a complete list of bugs fixed in release 1.12.

Enhancements in version 1.12.2

  • Support for the SOLUTIONTIME keyword was added to the Tecplot reader. Values specified for this keyword will appear as the 'time' value for plots.
  • The UNIC database reader was enhanced to support several new element types. The new element types consist of triangles, quadrilaterals, tetrahedra, pyramids, wedges and hexahedra ranging from linear to 20th order. The higher order elements are decomposed into linear elements when they are read in.

Bugs fixed in version 1.12.2

  • A bug was fixed with the Tube operator that could cause it to crash if the data had previously had an operation applied that removed some of the cells, such as the Subset or Threshold operators.
  • A Pseudocolor plot with the PersistentParticles operator now has the correct data extents.
  • A bug reading integer data from the Silo plugin, introduced in version 1.12.1, was fixed.
  • The ParallelCoordinates plot now works well with FastBit-accelerated indices and expression data.
  • The PersistentParticles operator now works with expression data.
  • The PersistentParticles operator now works with vector data.
  • The PersistentParticles operator no longer causes a parallel hang.
  • The PersistentParticles operator now issues a warning if it is not the first operator in a pipeline.
  • The VizSchema reader now supports external file links (allowing data spread across multiple h5 files).
  • The VizSchema reader now supports polygons and polyhedra.
  • A bug in the Silo plugin where block-structured decomposition and mrgtree objects did not play well together was fixed.
  • A problem where HDF5-based plugins would wind up terminating the HDF5 library and/or freeing global HDF5 library state out from underneath currently opened files was addressed.
