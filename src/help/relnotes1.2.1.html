VisIt 1.2.1 Release Notes

Welcome to VisIt's release notes page. This page describes the important enhancements and bug-fixes that were added to this release.

Features added in version 1.2.1

  • VisIt's expression code has been completely revamped so that all expression constructs such as constants, vectors, and user-defined functions work properly.
  • VisIt has been enhanced so it automatically uses scalable rendering mode for large datasets.
  • Pick can now return values for expression variables.
  • VisIt now has conditional expressions such as: if, and, or, which allow you to create sophisticated expressions.
  • The Clip operator has been rewritten so it is 15 to 100 times faster than the older version.
  • VisIt now has better support for vector expressions, including having vector dot and cross functions.
  • VisIt has a new BoxLib database reader plugin.
  • VisIt now has more glyph types for point meshes.
  • VisIt now recognizes .curve files so that they can be plotted by the Curve plot.
  • VisIt has a new matvf expression that can extract the material volume fractions from a material and create a scalar field so volume fractions can be plotted.
  • VisIt has support for parallel Plot3D files.
  • The vertex normals filter is faster.
  • Changing preference to orthogonal slice in the Slice operator window changes the origin type.
  • VisIt's filename sorting code has been changed to correctly sort grouped filenames that contain numbers.
  • VisIt's Pick window now has a toggle that allows the window to be shown automatically when new pick results are sent to the GUI.

Bugs fixed in version 1.2.1

  • Pick does not work on expression variables.
  • VisIt crashes when material selecting a volume plot when running in parallel.
  • The Subset plot, in conjunction with the Clip operator, can generate surfaces with bad normals.
  • All curves appear to be regenerated when a new curve is added.
  • VisIt appeared to hang when starting to generate a plot on white.
  • The Host profiles window causes the GUI to crash with certain config settings.
  • VisIt crashes when given an invalid expression.
  • The viewer sometimes crashes when the Engine selection window is active.
  • Empty functions in expressions crash the viewer.
  • 2D vector plot with zero y-component still creates vertical arrows.
  • Must click "Apply" too often in the Host profiles window.
  • There is an extra print statement somewhere in the scalable rendering code.
  • The Point tool has a swarm of extra points that move when the tool is moved.
  • If software volume rendering crashes the compute engine then the viewer also crashes.
  • Expressions do not clean up unused variables in the compute engine.
  • Axis labeling off with a Djehuty dataset.
  • The MDServer needs an interface to create expressions.
  • Rendering attributes are not setting set in the compute engine when in scalable rendering mode.
  • VisIt locks up when playing animations in Windows.
  • VisIt has problems reading certain VTK files.
  • Errors for invalid expressions should be made more clear.
  • Save movie does not work on Windows.
  • No -v flag passed to mpeg_encode from save movie.
  • The Box operator does not correctly handle thin boxes.
  • When the Reflect operator is in 2d mode, it shouldn't have +z on the original input quadrant.
  • Restore sessions doe not clear out saved views.
  • Saving sessions with multiple windows adds extra plots.
  • Save session now enforces a .session extension on session files.
