VisIt 1.9.1 Release Notes

Welcome to VisIt's release notes page. Release 1.9.1 is primarily a bug fix release to release 1.9. Look here to see a complete list of bugs fixed in release 1.9.

Bugs fixed in version 1.9.1

  • Added support for Curve plots of expressions derived from 1d flash data.
  • Fixed path-parsing problem on Windows systems that prevented plugins from loading on diskless configurations.
  • VisIt fails to locate and use movie templates when using the Save Movie Wizard.
  • The Spreadsheet plot is missing its top level menu in the MacOS X of VisIt.
  • The XDMF reader crashes the compute engine when reading data that uses the VxVyVz coordinate format.
  • VisIt does not use the compute engine configuration saved a session file when using a session file to generate a movie with "visit -movie".
  • Clicking on an imported color table in the Color Table window causes VisIt's GUI to crash.
  • VisIt truncates the username on Windows platform when 'Change username' option is chosen while connecting to remote machine.
  • VisIt's SimV1 reader required the GetMixedScalar callback function to be defined in a simulation in order for GetScalar to work. If the GetMixedScalar function is not defined then the SimV1 reader would crash.
  • A bug was fixed with Pseudocolor plot's of rectilinear grids with the Box operator applied.
  • The time varying Exodus file format reader no longer crashes when revisiting earlier time slices.
  • VisIt's CGNS reader now retrieves all relevant sections of an unstructured mesh instead of just returning the first section.
  • The Silo reader can now support more storage mechanisms for arbitrary polygons, specifically those that set the shapesize field to 0 and encode each polygon's number of nodes in the nodelist.
  • When existing path size was greater than 1024, the Windows installer overwrote the PATH with VisIt's install directory, instead of prepending to the PATH.
  • Allow multi-variable expressions to work with curves.
  • The Miranda reader had a problem reading some 2D data in the new format.
